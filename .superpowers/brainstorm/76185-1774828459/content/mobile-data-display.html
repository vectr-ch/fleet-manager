Mobile Data Display: Tables → ?

The fleet page has a 7-column table with hover-reveal action buttons. How should dense data adapt on small screens?

A

Card List

Each table row becomes a compact card. Primary info (name, status) is always visible. Secondary info (serial, firmware, base) stacks below. Action buttons are always visible at the card bottom — no hover needed.

Pros

  • Touch-friendly — no hover needed
  • Scannable, natural on mobile
  • Easy to tap into detail view

Cons

  • Takes more vertical space than table rows
  • Harder to compare across items (no column alignment)
B

Condensed Row List

Minimal rows showing only name + status. Tap a row to expand inline (accordion-style) revealing full details and action buttons. Keeps the list compact while making everything accessible.

Pros

  • Very compact — see more items at once
  • Progressive disclosure — details on demand
  • Closest to the table mental model

Cons

  • Requires tap to see anything beyond name/status
  • Accordion interaction can feel clunky
C

Horizontal Scroll Table

Keep the table as-is but wrap it in a horizontal scroll container. First column (name) stays sticky. User swipes to see more columns. Actions move to a tap-to-reveal menu (long-press or ··· button).

Pros

  • Minimal code change — wrap existing table
  • Column alignment preserved
  • Desktop and mobile use same component

Cons

  • Poor mobile UX — horizontal scroll is unintuitive
  • Hard to discover hidden columns
  • Doesn't solve the hover-action problem elegantly
