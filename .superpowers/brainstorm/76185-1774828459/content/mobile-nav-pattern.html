Mobile Navigation Pattern

The sidebar is fixed at 200px today — it needs to collapse on mobile. Which pattern fits VECTR best?

A

Bottom Tab Bar

iOS/Android-style fixed bottom tabs for the 4-5 most important views (Map, Fleet, Bases, Alerts, Overview). Sidebar disappears entirely on mobile. Settings and secondary pages accessible via a "More" tab or the user avatar menu.

Pros

  • Thumb-friendly, one-handed use
  • Always visible — no hidden navigation
  • Native mobile feel for field operators
  • Works great for quick status checks

Cons

  • Limited to ~5 items before needing overflow
  • Takes vertical space from content
  • Two different nav paradigms to maintain (sidebar + tabs)
B

Hamburger Drawer

The current sidebar slides in as an overlay drawer from the left, triggered by a hamburger icon in the topbar. All nav items stay in the same sidebar component — it just becomes toggleable on mobile.

Pros

  • One nav component for both breakpoints
  • Simpler implementation — same sidebar, different display mode
  • Full nav always accessible without item limits

Cons

  • Navigation is hidden — extra tap to reach anything
  • Hamburger menus have poor discoverability
  • Feels less native on mobile
C

Hybrid: Bottom Tabs + Drawer

Bottom tabs for the top 4 field-critical views (Map, Fleet, Bases, Alerts). A compact hamburger or "more" button opens a drawer for everything else (Overview, Missions, Telemetry, Audit, Settings). Best of both worlds.

Pros

  • Fast access to what field ops need most
  • Nothing is hidden — secondary items still reachable
  • Matches the "status + light ops" priority

Cons

  • Most complex to implement
  • Two mobile nav components to build and maintain
  • Needs careful item selection for bottom bar
