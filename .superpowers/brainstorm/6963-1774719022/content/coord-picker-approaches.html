Coordinate Picker — Where does the map live?

The picker needs enough space for a usable map + search. Three options for how it integrates with the base card.

A

Full Modal (Recommended)

A dedicated modal opens with a large map, search bar at top, and live coordinate readout. User clicks to place a pin, or types an address to geocode. Confirm button saves coords back to the form.

Pros: maximum map space, clean focus, matches existing modal patterns (create base, issue cert). Cons: takes user out of card context momentarily.

B

Inline Expandable (in card)

A "Choose on Map" button below the lat/lng inputs expands a small map inline within the card. Search bar sits above the map. Collapsible.

Pros: stays in context, no modal. Cons: very constrained space inside a card, map would be ~300px tall, awkward on mobile.

C

Slide-out Panel

A right-side panel (like fleet detail in the mock) slides out with the map. Stays visible alongside the card grid.

Pros: keeps card visible, good for comparing. Cons: complex layout shift, only works on wide screens, heavy for a simple coord pick.
